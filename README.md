# DEC vs k-means на Mall Customers

Небольшой проект, сравнивающий классические «k-средние» и «DEC (глубокая встроенная кластеризация)» в «Mall_Customers.csv».

## Что включено

- Предварительная обработка данных («CustomerID», «Пол», масштабирование функций).
- Базовая кластеризация с помощью «k-средних».
- Предварительная подготовка автоэнкодера и точная настройка DEC (`q`, `p`, `KL(P||Q)`).
- Сравнение методов по метрикам:
  - `Силуэт`
  - `Дэвис-Булдин`
  - `Калински-Харабаш`
- Кластерная визуализация по 2-м функциям:
  - `Годовой доход (тыс.$)`
  - `Оценка расходов (1-100)`

## Файл проекта

- `DEC_vs_kmeans_Mall_Customers.ipynb` — основной блокнот Colab с полным конвейером.

## Краткий результат текущего запуска

- В этом наборе данных «k-средние» превзошли «DEC» по основным внутренним показателям.
- В проверке с несколькими исходными значениями (начальные числа «5») «k-средние» также были более стабильными.
